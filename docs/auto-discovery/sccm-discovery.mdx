# SCCM Discovery

As of 18.11.00, SCCM Discovery is integrated into our main discovery jobs. Microsoft SCCM (System Center Configuration Manager or MECM, Microsoft Endpoint Configuration Manager), is a comprehensive management platform to manage and deploy software, applications, updates, and operating systems in an enterprise environment. SCCM provides administrators with a centralized and unified solution to efficiently manage a large number of devices, both physical and virtual, running various versions of Windows, macOS, Linux, etc.

If you are already using SCCM in your environment, the Device42 SCCM integration can automatically sync the hardware and software inventory (CI) data to Device42. SCCM discovery that can discover devices, OS, CPU, memory, network and software from SCCM.

SCCM Discovery can be configured with either WDS or WinRM and database discovery can be used alongside WDS and WinRM to provide a comprehensive view of the environment.

## SSCM Database Discovery on Windows Targets

For discovery to return detailed information, you will require read permissions to the following system views:

<table>
  <tbody>
    <tr>
      <td width="288">
        <ul>
          <li>INFORMATION_SCHEMA.TABLES</li>
          <li>v_GS_COMPUTER_SYSTEM</li>
          <li>v_GS_X86_PC_MEMORY</li>
          <li>v_GS_Add_Remove_Programs</li>
          <li>v_R_System</li>
        </ul>
      </td>
      <td width="288">
        <ul>
          <li>v_GS_PC_BIOS</li>
          <li>v_GS_PROCESSOR</li>
          <li>v_GS_NETWORK_ADAPTER_CONFIGURATION</li>
          <li>v_GS_Add_Remove_Programs_64</li>
        </ul>
      </td>
    </tr>
  </tbody>
</table>

When creating or editing a job, check the **Collect with database server** option and provide the necessary discovery target credentials.
